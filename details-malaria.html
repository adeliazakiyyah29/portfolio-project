Speech Analytics for Smarter Support

  • Problem Definition

    • Customer service departments often receive dozens or even hundreds of calls daily. However, tracking metrics such as call volume, call duration, customer satisfaction, and repeat callers is still limited in many systems.

    This project proposes an automated speech analytics solution that:

    • Transcribes call audio to text using OpenAI Whisper
    • Analyzes sentiment and emotions using NLP models
    • Tracks key insights such as call frequency, duration, and customer satisfaction trends
  • Data Input
    • Customer service call recordings
    • Transcribed using Whisper for speech-to-text conversion
  • Exploratory Data Analysis (EDA)
    • Call volume per agent per day
    • Sentiment distribution (positive, negative, neutral)
    • Total handle time
    • Repeat caller detection
  • Data Preprocessing
    • Text cleaning (removal of stopwords and punctuation)
    • Speaker diarization to label speakers (e.g., speaker_0, speaker_1)
  • Modeling

    • For modeling, use pre-train sourced from HuggingFace as follows:

    • Sentiment Analysis

      • Fine-tuned RoBERTa model on Indonesian language sentiment labels (positif, negatif, netral)

    • Emotion classification

      • Fine-tuned RoBERTa model to classify emotions (angry, happy,neutral.)

  • Evaluation and Result

    • Output include :

    • Transcription results
    • Sentiment scores
    • Emotion classification scores
