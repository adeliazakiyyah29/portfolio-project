Speech Analytics for Smarter Support

  • Problem Definition

    • setiap harinya perusahaan layanan pelanggan menerima puluhan atau mungkin ratusan panggilan telepon setiap hari. Tapi proses pelacakan terkait jumlah panggilan, dan apakah customer puas dengan pelayanan yang diberikan masih terbatas.

    Bagaimana jika ada sistem yang dapat melacak berapa lama mereka telpon, total handle time, berapa kali customer telpon dan apakah bantuan call agent membantu?

    Solusi yang diusulkan :

    • Speech recognition

      • menggunakan model speech to text menggunakan whisper

    • Natural Language Processing

      • 1. Deteksi sentimen

      2. Analisis emosi pelanggan

  • Data Input

    • Rekaman audio percakapan pelanggan menggunakan OpenAI Whisper

  • Exploratory Data Analysis (EDA)
    • Total jumlah panggilan perhari setiap agent service
    • Identifikasi emosi (positif, negatif, netral)
    • Total handle time
    • Identifikasi pelanggan yang berkali-kali telepon
  • Data Preprocessing
    • Text Cleaning (hapus kata sisipan dan tanda baca)
    • Speaker Diarization (identifikasi siapa yang berbicara pada rekaman audio dan melabelkan menjadi speaker0, dan speaker1)
  • Modeling

    • Untuk modeling menggunakan pre-train yang bersumber dari Hugging face sbb:

    • Sentiment Analysis

      • Model RoBERTa yang di-fine-tune untuk klasifikasi sentimen (positif, negatif, netral) dalam bahasa Indonesia.

    • Emotion classification

      • Model RoBERTa yang di-fine-tune untuk mendeteksi emosi seperti: marah, senang, sedih, takut, netral dalam bahasa Indonesia.

  • Evaluation and Result

    • Didapatkan hasil berupa score untuk sentiment analys dan juga emotion classification. Score merupakan logit yang diproses menggunakan fungsi softmax dengan range index probabilitas dari 0-1. Kemudian hasil transkripsi serta score ini akan di simpan ke dalam database SQL Server Management Studio
